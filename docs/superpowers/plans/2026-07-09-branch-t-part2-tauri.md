# Standalone bridge — Branch T, Part 2: Tauri shell + packaging + landing page

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Wrap the Part 1 `bridge-core` engine in a Tauri v2 tray/menu-bar app that a non-technical user downloads and runs — guided ytmdesktop onboarding, auth-code surfacing, launch-at-login — then ship it via GitHub Releases and rewrite the landing page around the no-toolchain path.

**Architecture:** Add a `src-tauri` crate to the existing `app/` workspace. Its `setup` spawns `bridge_core::run(tx)` on Tauri's tokio runtime and forwards each `BridgeEvent` to the webview via `app.emit("bridge-event", …)`. A static (no-bundler) frontend at `app/ui/` renders the state machine and the matching call-to-action. The tray icon reflects `BridgeState`; auth codes also fire an OS notification; a command toggles launch-at-login. CI builds Windows + macOS installers on a runner matrix and attaches them to the same GitHub Release as the firmware. The board wire contract and `bridge-core` behavior are unchanged — this is a shell around the engine.

**Tech Stack:** Rust, Tauri v2 (`tauri`, `tauri-plugin-notification`, `tauri-plugin-autostart`, `tauri-plugin-opener`), static HTML/CSS/JS frontend (no Node bundler), `tauri-action` (CI), `serde`.

## Global Constraints

Every task's requirements implicitly include these.

- **Toolchain reality (READ FIRST — gates the whole plan):** Tauri on **Windows requires the MSVC toolchain + WebView2 runtime**; the GNU toolchain Part 1 used is **not** supported by Tauri/wry/tao. This machine's MSVC currently fails to link (MSYS2 `link.exe` shadow, no VS C++ Build Tools — see the `rust-toolchain-use-gnu-not-msvc` note). Therefore **local** Tauri builds require installing **Visual Studio C++ Build Tools** (which also fixes the MSVC linker) OR doing Tauri work on macOS. **CI is unaffected** — GitHub's `windows-latest`/`macos-latest` runners have MSVC + WebView2. Task 1 is a spike that establishes a working local Tauri build (or explicitly decides "CI-only builds, UI iterated on macOS"). `bridge-core`/`bridge-cli` keep their `rust-toolchain.toml` gnu pin; the Tauri crate must build under MSVC — do **not** let the workspace's gnu pin force the Tauri crate onto gnu (pin per-directory or remove the workspace pin and pin the two library crates).
- **`bridge-core` public API consumed (from Part 1, on `main`):** `bridge_core::run(events: tokio::sync::mpsc::UnboundedSender<BridgeEvent>) -> anyhow::Result<()>`; `enum BridgeEvent { State(BridgeState), AuthCode { code: String }, NowPlaying(NowPlayingVm), Log(String) }` (Debug + Clone); `BridgeState` (serde, kebab-case: `starting`/`ytmd-not-found`/`not-authorized`/`waiting-for-board`/`board-connected`/`ytmd-disconnected`); `NowPlayingVm` (serde, the snake_case vm keys); `Playback`.
- **State → UI mapping (from the design doc):**
  - `ytmd-not-found` → "YouTube Music Desktop isn't running." + **[Download ytmdesktop]** + [How to enable Companion Server].
  - `not-authorized` → large **auth code** + "Click **Allow** in ytmdesktop within 30s" (also an OS notification).
  - `board-connected` → "✓ Connected — your board is live." + launch-at-login toggle.
- **Non-goals (YAGNI — do NOT build):** auth-update/auto-update; code signing (accept SmartScreen/Gatekeeper, document the bypass); in-app install of ytmdesktop (detect + link only); any settings beyond launch-at-login; packaging the firmware flasher into the app (flashing stays in the browser).
- **Distribution:** GitHub Releases, alongside the firmware image on `v*` tags. **Extend the existing `release.yml`; do not add a parallel release pipeline.**
- **Don't touch the board wire contract or `bridge-core` behavior.** The only permitted `bridge-core` change is deriving `Serialize` on `BridgeEvent` (Task 2) so it can cross the Tauri IPC boundary.
- **Coordinate with the in-flight `re-auth on token revocation` follow-up** (separate session touching `bridge-core`): it will add re-auth that re-emits `BridgeEvent::AuthCode`. The frontend's `not-authorized`/auth-code handling (Task 5/7) must treat a repeat `AuthCode` as a refresh, not assume it fires once.

let handle = app.handle().clone();
let (tx, mut rx) = tokio::sync::mpsc::unbounded_channel::<bridge_core::BridgeEvent>();
tauri::async_runtime::spawn(async move {
    if let Err(e) = bridge_core::run(tx).await {

## Task 5: Guided status window (frontend renders the state machine + CTAs)

- [ ] **Step 1: Render each state with its CTA**

`main.js` listens for `bridge-event` and switches on `payload.type`/`payload.data`:
```js
const { listen } = window.__TAURI__.event;
const { invoke } = window.__TAURI__.core;
const opener = window.__TAURI__.opener; // opener plugin

const app = document.getElementById("app");

function renderState(state) {
  switch (state) {
    case "ytmd-not-found":
      return view({
        icon: "🔌",
        title: "YouTube Music Desktop isn’t running",
        body: "Start ytmdesktop and enable its Companion Server.",
        actions: [
          { label: "Download ytmdesktop", url: "https://ytmdesktop.app" },
          { label: "How to enable Companion Server", url: "https://github.com/ytmdesktop/ytmdesktop#companion-server" },
        ],
      });
    case "not-authorized":
      return null; // handled by the auth-code event (Task 7)
    case "waiting-for-board":
      return view({ icon: "🛰️", title: "Waiting for your board…", body: "Authorized. Power on the board on the same Wi-Fi." });
    case "board-connected":
      return connectedView();
    case "ytmd-disconnected":
      return view({ icon: "⚠️", title: "Lost YouTube Music Desktop", body: "Reconnecting…" });
    default:
      return view({ icon: "⏳", title: "Starting…", body: "" });
  }
}

listen("bridge-event", ({ payload }) => {
  if (payload.type === "state") app.replaceChildren(renderState(payload.data));
  else if (payload.type === "auth-code") showAuthCode(payload.code);   // Task 7
  else if (payload.type === "now-playing") updateNowPlaying(payload.data);
});
```
